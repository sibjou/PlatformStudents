# Data Layer: PlatformStudents-Core-Architect

Данный слой отвечает за управление всеми данными приложения, включая локальное хранение, кэширование и миграции.

## Архитектура
Слой данных разделен на три части:
1. **Models**: Plain Old Dart Objects (PODO) с поддержкой сериализации в Hive.
2. **Local**: Низкоуровневые сервисы для работы с БД (Hive), настройками (SharedPreferences) и файловой системой.
3. **Repositories**: Высокоуровневая абстракция, предоставляющая интерфейс для доступа к данным остальной части приложения.

## Кэширование и TTL
В приложении реализовано два типа кэша:
- **TaskCacheRepository**: Кэширует JSON-ответы от API во внутреннюю БД Hive. По умолчанию время жизни (TTL) — 2 часа.
- **ImageCacheRepository**: Кэширует изображения во временную директорию устройства. Использует MD5-хеширование URL для именования файлов. Поддерживает TTL и автоматическую очистку при превышении лимита в 50 МБ.

## Миграции
Сервис `MigrationService` управляет версионностью локальной БД:
- **v1 → v2**: Добавление поля `status` в модель `Assignment`.
- **v2 → v3**: Добавление опционального поля `hashRes` для хранения результата выполнения.

## Резервное копирование
`BackupService` позволяет экспортировать все данные из Hive в единый JSON-файл и восстанавливать их из этого файла.

Схема Базы Данных (Database Schema)
Hive Boxes (NoSQL) 
1. assignments (Box<Assignment>)
Хранит информацию о выданных заданиях.
| Поле      | Тип     | Описание                                |
| :-------  | :------ | :----------------------------------     |
| id        | String  | Уникальный ID задания                   |
| studentId | String  | ID студента                             |
|startTaskId| int     | Начальный ID задачи в системе           |
| hashUsl   | String  | Хеш условий задания                     |
| createdAt | DateTime| Дата создания                           |
| status    | String  | Статус (pending, inProgress, completed) |
| hashRes   | String? | Хеш результата (может быть null)        |

2. task_cache (Box)
Хранит сырые ответы от Task API.

Key: String (ID задачи).

Value: Map { data: String (JSON), ts: String (ISO-дата кэширования) }.

3. settings (Box)
Служебная информация и настройки.
SharedPreferences (KeyValue)
Ключ             Тип                     Описание
db_version       int            Текущая версия схемы данных
teacher_id       intID          текущего преподавателя
test_progress   List<String>    Массив ответов текущего теста
recent_hashes   List<String>    Список последних использованных хешей
